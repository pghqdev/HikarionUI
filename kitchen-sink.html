Elevation

Five rungs, --elevation-0 through --elevation-4, each meaning a material state rather than a size. A crisp contact shadow plus one ambient layer, inked from the theme’s accent so a theme swap re-lights the whole ladder. --shadow-sm, --shadow and --shadow-lg alias rungs 1, 2 and 4.

Context-aware cards

An <article> is the framework's query container, so what is nested in a card adapts to the column it actually sits in — not to the viewport. Wide enough and a leading <figure> floats into a media column; narrow and it stacks, a [data-nav] run turns into a rail, and an action row takes the full width. No hook, no size variant, no breakpoint: the same markup twice. Compact shrinks the card's padding, so the content box is wider and the media column arrives slightly sooner — the labels and copy hold their size.

Switch

Dropdown & action menu

Native popover — the browser owns open, light-dismiss, and focus. Pinned under its trigger with CSS anchor positioning, no JS. Rows are plain buttons and links: a destructive row is data-variant="danger", an inert row is aria-disabled (still focusable and announced), and a shortcut hint is a real <kbd>. With hikarion.js loaded it is an APG menu: ↑/↓, Home/End and first-character typeahead move between rows, Esc or Tab closes it. Without the script it stays an honest disclosure — Tab moves between rows and no menu roles are claimed. Compact tightens the rows and the trigger; the labels hold their size.

Button group & split button

[data-button-group] fuses adjacent <button>s into one control: one hairline per seam, outer corners only. A split button is the same group with a [popovertarget] half, so the chevron, panel and light-dismiss all come from [data-menu]. The group is a single row and does not wrap. Compact tightens the buttons themselves; the seams and the corner radius are unchanged.

Tooltip

CSS-only [data-tooltip] — hover or focus to reveal. It waits --hk-tooltip-delay (0.4s) before showing and lingers --hk-tooltip-delay-out after, so the pointer can travel onto the bubble without dismissing it (WCAG 2.2 1.4.13 hoverable). It cannot be dismissed with Esc, nor revealed from the keyboard on a non-focusable element — for that, use popover="hint": a real element toggled by a [popovertarget] button. Esc, light-dismiss and focus return come from the browser where popover="hint" is implemented (not in any shipping Safari today); elsewhere it falls back to the manual state and hikarion.js supplies Esc and outside-click itself. Compact tightens the bubble's padding and text; the delays are density-invariant.

Toast

Transient feedback via Hikarion.toast() — auto-dismisses (the timer pauses while the toast is hovered or focused), or close it with the ✕. Stacks bottom-end in the top layer, so toasts clear an open modal; the stack is capped at the viewport and clips its oldest members rather than running off-screen. The danger tone announces assertively, every other tone politely. No density pair here, deliberately: the region is a fixed top-layer element on <body>, so it reads :root density, not a data-density container in the page.

Select & searchable input

No hook — <select>, <optgroup>, select multiple and <input list> + <datalist> carry every meaning themselves. The searchable case is a real native combobox: typed filtering, keyboard and screen-reader behaviour owned by the browser, zero JS. Compact tightens the field chrome and the list-box row inset; row height and option text are the browser's and hold their size.

Date & time fields

input[type="date"], time, datetime-local, month, week — the field chrome every input gets, plus the picker button repainted as the same chevron <select> uses, so it survives a dark theme and promises the same drawer. The platform picker is the picker; Hikarion ships no calendar. Compact tightens the field padding like every other field; the segment row and its digits hold their size.

File upload previews

A plain <ul> placed after a [data-dropzone] label — not inside it, because the invisible file input covers the card and would swallow the clicks. Each <li> is a preview slot holding an optional thumbnail and its filename. aria-live="polite" on the filename slot announces the selection; no role="status", that hook renders an alert box. Compact tightens the card, gaps and slot width; filenames hold their size.

Progress & meter

Native <progress> and <meter> — bars in the accent; <output> shows a computed value.

Circular progress

The spinner with role="progressbar" becomes determinate — same hook, no new vocabulary, no JS. --hk-value paints the fill; CSS can’t read a number out of aria-valuenow, so the value is given twice.

Meter thresholds & range

A meter's empty track is the same hairline as <progress>, so a gauge at 0 reads as empty; the low / high / optimum regions carry all the colour. Compact thins the meter bar and the range groove and leaves the drag handle alone: the slider row stays 24 px tall in both densities, so the target never shrinks below the WCAG 2.2 floor.

Scroll progress

[data-scroll-progress] — an empty element that becomes a hairline reading-progress bar across the top of the viewport, driven by animation-timeline: scroll(root block). It is live on this page right now: scroll and watch the top edge. Purely presentational, so it carries aria-hidden and never takes pointer events. Density does not touch it — 3px is the thinnest honest hairline — and there is no per-instance variant, because a document has exactly one reading position.

Where scroll() is unsupported, and whenever prefers-reduced-motion: reduce is set, the element paints nothing at all: no bar, not a slower bar. A progress bar that cannot follow the scroll must not draw a position it does not know.

<div data-scroll-progress aria-hidden="true"></div>
